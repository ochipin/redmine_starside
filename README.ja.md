# Redmine Starside

[English](README.md) | **日本語**

**[Starlight](https://starlight.astro.build/) 風のバナー・ステップリスト・タブ・チェックボックス・技術スタックバッジを Redmine の Wiki やチケットに追加する、小さなプラグインです。**

素の Redmine の Wiki はすぐに単調になりがちです。すべてが同じ見た目で、注意書きを目立たせたり手順を分かりやすく並べたりするのが難しい。このプラグインは、見栄えのする注意喚起バナー・番号付きステップリスト・タブ表示・技術スタックバッジを追加する Wiki マクロをいくつか提供し、さらにチェックボックス記法を自動変換してタスクリストを自然に書けるようにします。見た目は Astro の Starlight ドキュメントテーマから拝借しています。

バッジ以外はすべてローカルで描画され、外部 CDN や API への通信はありません。そのため、**主要機能はインターネットに接続できない閉域環境（オンプレ / 社内ネットワーク）でもそのまま動作します。** バッジマクロだけは例外で、既定では [shields.io](https://shields.io/) を参照しますが、完全な閉域向けに self-host した shields インスタンスへ向けることもできます（[バッジ](#バッジ)を参照）。

## 機能

**注意喚起バナー**
`tip` / `note` / `warning` / `danger` の 4 種類のバナー。それぞれ色とアイコンが異なり、補足や注意を目立たせられます。タイトルは省略可能で、本文は Textile / Markdown として解釈されるため、バナーの中で強調・リンク・リストなどがそのまま使えます。

**ステップリスト**
通常の番号付きリストを、丸数字と縦線でつながった手順表示に変換します。順序が重要なインストール手順や運用手順（ランブック）に向いています。

**タブ**
複数の内容をタブで切り替えて表示します。OS ごとのコマンドや、同じ内容の Markdown / Textile 版の併記などに便利です。マウス操作のほかキーボード（矢印キー / Home / End / Enter / Space）でも操作でき、アクセシビリティのための ARIA ロールも付与しています。

**チェックボックス記法**
本文に `[ ]` や `[x]` と書くと、自動でチェックボックス（☐ / ☑）に変換されます。入力中のプレビューにも追従し、コードブロック内の記法は変換対象外なのでコード例はそのまま残ります。Textile では `[X]` がリンクに変換されてしまうことがありますが、その場合も自動で救済してチェック済み表示にします。

**技術スタックバッジ**
`{{badge(docker)}}` のように書くと、ブランドカラーとロゴ付きの shields.io 形式のバッジを表示します。OS・言語・データベース・コンテナ / CI ツールなど約 90 個のキーを内蔵しており、`{{badge(redmine, 6.1+)}}` のようにバージョンを併記することもできます。色やラベルの変更、新しいバッジの追加は、プラグインの設定画面から行えます（コード変更は不要）。バッジ画像の生成元は既定で shields.io ですが、閉域向けに別のホスト（リバースプロキシ配下のルート相対パスを含む）へ向けることもできます。

## 動作確認環境

- Redmine 6.1（Propshaft 環境）
- テキスト書式：Markdown / Textile の両方に対応（「管理 > 設定 > 全般」で選択）
- 主要機能は外部依存なし・完全オフラインで動作（バッジは任意で shields.io を利用 — 「バッジ」を参照）

## インストール

### Step 1: プラグインを配置

`redmine_starside` ディレクトリを Redmine の `plugins/` 配下に置きます。

```
<REDMINE_ROOT>/plugins/redmine_starside/
```

### Step 2: Redmine を再起動

Web サーバ（Puma / Passenger など）を再起動します。マイグレーションは不要です。

`init.rb` が ViewHook を登録しており、`starside.css` と `starside.js` を全ページの `<head>` に自動で読み込みます。そのため、**アセットの手動コピーやシンボリックリンクの作成は不要です。**

### 動作確認

Wiki またはチケットの編集画面を開き、下記のようなバナーを書いてプレビューします。色付きの枠が表示されれば成功です。

```
{{note(こんにちは)
動きました。
}}
```

## 使い方

### バナー

注意書きや補足を目立たせる枠です。引数にタイトル、本文にマクロのブロック内テキストを書きます。本文は Textile / Markdown として解釈されます。

```
{{note(補足)
ここに本文を書きます。**強調** やリンクも使えます。
}}
```

タイトルを省略すると、種類名（Tip / Note / Warning / Danger）が既定で表示されます。

```
{{warning
タイトルを省略した場合の例です。
}}
```

本文を引数の 2 番目として渡すこともできます（短い 1 行向け）。

```
{{tip(ヒント, 一行だけの補足)}}
```

| マクロ | 用途 | 色 |
|--------|------|----|
| `tip` | ヒント・豆知識 | 紫 |
| `note` | 補足・備考 | 青 |
| `warning` | 注意 | 黄 |
| `danger` | 危険・警告 | 赤 |

### ステップリスト

本文に通常の番号付きリストを書くと、つながった番号付きの手順表示になります。

```
{{step
# 最初の手順
# 次の手順
# 最後の手順
}}
```

### タブ

引数にタブ名をカンマ区切りで指定し、本文を `+++` で区切ります。

```
{{tabs(タブA, タブB, タブC)
タブ A の内容です。
+++
タブ B の内容です。
+++
タブ C の内容です。
}}
```

タブ名を省略した分は `Tab 1`, `Tab 2`... と自動で補完されます。

### チェックボックス記法

Wiki 本文・プレビュー・チケットのコメント内で、以下の記法が自動でチェックボックスに変換されます。

| 記法 | 表示 |
|------|------|
| `[ ]` | ☐（未チェック） |
| `[x]` `[X]` `[*]` | ☑（チェック済み） |

```
[ ] まだ終わっていないタスク
[x] 完了したタスク
```

コードブロック（`<pre>` / `<code>`）内の記法は変換対象外なので、コード例として書いた `[ ]` はそのまま残ります。

### バッジ

技術スタックのバッジをインラインで表示します。第 1 引数にキー、任意の第 2 引数にバージョンを指定します。

```
{{badge(linux)}}
{{badge(redmine)}}
{{badge(redmine, 6.1+)}}
```

バージョン末尾の `+` は `.*` に変換されます（`6.1+` → `6.1.*`）。キーは大文字小文字を区別せず、いくつかのエイリアス（`k8s` → Kubernetes、`golang` → Go など）も用意しています。

OS・ディストリビューション、言語、データベース、コンテナ / CI / IaC ツール、シークレット / 監視ツール、Google Workspace、汎用アイコン（`settings`・`maintenance`・`bug`・`network`）など、約 90 個のキーを内蔵しています。利用可能なキーの最新の一覧は、色とプレビュー付きで設定画面に表示されます。

**カスタマイズ**
「管理 > プラグイン > Redmine Starside > 設定」から、以下が行えます。

- 内蔵バッジの色の変更
- 独自バッジの追加（既定にないツールも追加可能）
- バッジ画像の生成元（`badge_base_url`）の設定

保存されるのは変更点だけなので、プラグイン更新で既定色が改善された場合、カスタマイズしていないバッジには自動で反映されます。

**閉域環境**
既定では `https://img.shields.io` に対してバッジを生成します。完全にオフラインで運用するには、self-host した [shields](https://hub.docker.com/r/shieldsio/shields) インスタンスを立て、`badge_base_url` をそれに向けます。絶対 URL（`https://localhost/shields`）でもルート相対パス（`/shields`）でも指定でき、リバースプロキシ配下ではルート相対パスを推奨します。バッジが表示中ページと同一オリジンとして解決されるため、自己署名証明書の影響を受けません。

> 他のマクロと異なり、`badge` はバッジ画像ホストを指す `<img>` を生成します。リバースプロキシ配下でルート相対の `badge_base_url` を使えば同一オリジンに収まり、オフライン動作の前提を保てます。

## アンインストール

1. （任意）プラグインが保存した設定を削除します。
   ```
   bundle exec rake redmine_starside:uninstall_settings RAILS_ENV=production
   ```
   バッジ設定画面を使った場合のみ必要です。設定行が残っていても無害です。
2. `plugins/redmine_starside/` を削除
3. Redmine を再起動

素の Wiki 表示に戻ります。
